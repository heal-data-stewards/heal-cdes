## How this compares to the earlier `2026apr20` run

The previous comparison ([`reports/2026apr20-from-2026jan20`](../2026apr20-from-2026jan20/report.md))
showed 2026apr20 vs 2026jan20. That comparison had:
- **195** common files, **40** only in apr20, **38** only in jan20.

This comparison (2026apr23-3 vs 2026jan20) shows:
- **223** common files, **40** only in apr23-3, **10** only in jan20.

The file set "only in 2026jan20" shrank from **38 → 10**, meaning the
2026apr23-3 pipeline recovered **28 CRFs** that were previously missing
relative to the jan20 baseline. The 40 files only in 2026apr23-3 are
identical to those in 2026apr20 — no new CRFs were added in this run, but
none were lost either.

## Summary of differences

The 10 remaining files only in 2026jan20 break down into two groups:

| --- | --- |
| `promis-29-profile` | `promis-29-profile-v2.1` |
| `promis-43-profile` | `promis-43-profile-v2.1` |
| `promis-propr-292-profile` | `promis-292-profile-v2.1-propr` |
| `promis-ability-to-participate-in-social-roles-and-activities-4a` | `promis-ability-to-participate-in-social_roles-and-activities-4a` |
| `promis-ability-to-participate-in-social-roles-and-activities-8a` | `promis-ability-to-participate-in-social_roles-and-activities-8a` |
| `promis-sleep-disturbance-6a` | `sleep-disturbance-6a` |

The `social_roles`/`social-roles` pair again highlights that 2026jan20 used
underscores where dashes belong. The `sleep-disturbance-6a` → `promis-sleep-disturbance-6a`
rename adds the PROMIS namespace prefix. Version suffixes (`-v2.1`, `-v2.1-propr`)
were dropped and slug order normalized.

**Genuinely absent from 2026apr23-3 (4 files):**

- `csq-24.dug.json`
- `pcs-child.dug.json`
- `pcs-parent.dug.json`
- `whodas-2.dug.json`

Note: `whodas-12-item.dug.json` is present in both; `whodas-2` is a
different (shorter) version of the same instrument. `csq-24`, `pcs-child`,
and `pcs-parent` are wholly absent. Worth checking the mapping CSVs.

### 2. Shared files are overwhelmingly equivalent

160 of 223 shared files (72%) are `trivial_only` — functionally identical
after normalization. The noise is the same three sources as before:

- `drupal_id` = `null` in 2026apr23-3 vs numeric strings in 2026jan20 (Drupal
  IDs not populated in the current pipeline).
- `Heal_CDE_2026-04-06T130104` vs `Heal_CDE_2026-01-05T130102` — the REDCap
  export timestamp baked into `_sources` paths.
- NIH CDE upload month in document URLs:
  `/sites/default/files/CDEs/<YYYY-MM>/`.

### 3. Demographic CDEs: genuine data-source divergence

The largest churn is in `pediatric-demographic.dug.json` (5+8+38 = 51 records
affected) and `adult-demographics.dug.json` (3+5+30 = 38 records affected).
The pattern is identical to what the apr20 comparison found:

| Present in 2026apr23-3 | Present in 2026jan20 |
| --- | --- |
| `EDULVL` | `EDULEVEL` |
| `ETHNIC` | _(absent)_ |
| `PAINDUR_acute` | _(absent)_ |
| _(absent)_ | `EDULEVEL` |
| _(absent)_ | `GENIDENT` |
| _(absent)_ | `GENIDENTOTH` |
| _(absent)_ | `HI_LA` |
| _(absent)_ | `MENA` |
| _(absent)_ | `HI_LA_p` _(pediatric only)_ |
| _(absent)_ | `MENA_p` _(pediatric only)_ |

This is a fundamental data-source difference, not a pipeline bug. The 2026jan20
data came from HEAL CDE Excel files which had newer gender-identity
(`GENIDENT`, `GENIDENTOTH`) and race/ethnicity (`HI_LA`, `MENA`) variables
and had renamed the education level field to `EDULEVEL`. The 2026apr23-3
pipeline pulls from the NIH CDE Repository API, which may not yet reflect
these updates. **Action item**: check whether the NIH CDE Repo has published
the updated demographic CDEs; if not, this gap will persist.

The 38/30 changed records in these files also reflect widespread rewrites of
`metadata.instructions`, `metadata.question_text`, `metadata.short_description`,
and `metadata.permissible_values` — consistent with a full CDE-content update
in the jan20 source that the NIH Repo hasn't yet received.

### 4. CRF-specific content changes of note

**Generalized Anxiety Disorder 7 (GAD-7)**: The two GAD-2 subscale variables
have full names in 2026apr23-3 (`GAD2FeelNervScale`, `GAD2NotStopWryScale`)
but are truncated in 2026jan20 (`GAD2FeelNervScl`, `GAD2NotStopWryScl`). The
full names in apr23-3 are the fix; 2026jan20 was truncating at 15 characters.
Study mappings also differ: `HDP00619` maps to `gad701` in 2026apr23-3
(on the first variable) but to `gad704` in 2026jan20 (on the second variable) —
one of these is a misassignment worth verifying. 2026jan20 has richer categories
(`Adult Acute`, `Adult Chronic`, `All Core`, `Pediatrics`, `Supplemental`);
2026apr23-3 has only `Supplemental`.

**Patient Global Impression of Severity (PGIS)**: Significantly restructured
in 2026apr23-3. The single `PGISSeverityScale` variable from 2026jan20 became
`PGIS7d` (patient self-report, explicit 7-day window) plus a new parent-proxy
variant `PGIS7dPP`. The CRF was renamed to "Patient Global Impression of
Severity (PGIS) - Chronic", and the question text was updated from "past week"
to "past 7 days" for consistency. This is a genuine content improvement in
2026apr23-3.

**Parent's Postoperative Pain Management (PPPM)**: 2026apr23-3 adds a new
variable `PPPMSF_score` (PPPM short-form total score). However, it also has
suspect permissible values on an existing variable — `"0": "1 = No"` and
`"1": "Yes"` vs the correct `"0": "Not at all"` / `"1": " A little"` in
2026jan20. The 2026apr23-3 version also gives a slightly different item count
(17 CDEs) and description from 2026jan20 (16 CDEs). **Potential data error**
in 2026apr23-3 worth investigating.

**Opioid MME**: Buprenorphine conversion factors differ:
- Sublingual film/tablet: 2026apr23-3 uses **38.8**; 2026jan20 uses **40**.
- Buccal film: 2026apr23-3 uses **0.039**; 2026jan20 uses **0.03**.

The apr23-3 values (38.8, 0.039) appear more precise and are consistent with
updated CDC guidance. The jan20 values (40, 0.03) are round-number approximations.
Also, a parenthesis typo in the hydrocodone and hydromorphone long-acting
formula strings (`(1 * ...` with mismatched parens) is present in 2026jan20
but fixed in 2026apr23-3.

**NIDA-ASSIST (both forms)**: 16 records changed in each — all study-variable
mapping refreshes (`metadata.study_variable_mappings`) plus new study
associations `HDP01288`. No structural CDE changes.

**Patient Health Questionnaire 9 (PHQ-9)**: 9 changed records, primarily
new study mappings (`HDP01492`, `HDP00982`, `HDP01105`, `HDP01061`) and
updated `_sources` on existing ones.

### 5. Study-mapping growth

Across ~30 files with single-record changes, the dominant pattern is new HDP
IDs appearing in `metadata.study_mappings` and `metadata.study_variable_mappings`.
Notable clusters:

- **HDP01629** added to `ascq-me-pain-episode-frequency-and-severity`,
  `ascq-pain-impact`, `promis-sleep-disturbance-4a`,
  `sickle-cell-disease-self-efficacy-scale`, `whoqol-bref`.
- **HDP01660** added to `aces`, `lotr`, `mbm`, `phq-m`,
  `promis-pediatric-pain-quality-sensory-8a`, `sni`.
- **HDP01543** and **HDP01672** appear across several pain and quality-of-life
  forms.

This is the expected result of a newer REDCap export with more study/CRF
associations than the one used for 2026jan20.

### 6. Category classification differences

~15 files have `metadata.categories` changes beyond those caused by study-mapping
additions. The 2026jan20 source tends to carry more explicit multi-value
category lists (e.g. `["Adult Acute", "Adult Chronic", "All Core", "Pediatrics",
"Supplemental"]`), while 2026apr23-3 often has a single or simpler list. This
is a systematic difference attributable to the data source and warrants a
targeted review if categories drive search/display behavior downstream.

## Summary

The 2026apr23-3 run represents a **substantial improvement** over the earlier
2026apr20 run relative to the 2026jan20 baseline: 28 previously-missing CRFs
are now present, reducing the jan20-only count from 38 to 10. Of those 10,
six are confirmed renames and only four (`csq-24`, `pcs-child`, `pcs-parent`,
`whodas-2`) are genuinely absent.

The main remaining divergences are structural: (a) demographic CDEs reflect a
data-source difference that the NIH CDE Repo may need to catch up on, (b)
category metadata is systematically sparser in the current pipeline, and (c)
a handful of CRF-specific content differences (PPPM permissible values, GAD-7
study mapping assignment) warrant spot-checking.

The large body of shared content (72% trivial-only, all normalized away cleanly)
confirms that the pipeline is producing correct output for the majority of CRFs.
