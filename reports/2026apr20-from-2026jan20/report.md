## Summary of differences

### 1. File set has shifted substantially

The two runs share **195** filenames but disagree on **78** (40 only in
2026apr20, 38 only in 2026jan20). Three patterns account for most of
these:

- **Slug / version renames** — same CRF, different filename. The following
  pairs are almost certainly renames, not additions/removals (see
  `tmp/rename_candidates.txt` for the Jaccard scoring):

  | 2026apr20                                                         | 2026jan20                                                         |
  | ----------------------------------------------------------------- | ----------------------------------------------------------------- |
  | `promis-29-profile`                                               | `promis-29-profile-v2.1`                                          |
  | `promis-43-profile`                                               | `promis-43-profile-v2.1`                                          |
  | `promis-propr-292-profile`                                        | `promis-292-profile-v2.1-propr`                                   |
  | `promis-ability-to-participate-in-social-roles-and-activities-4a` | `promis-ability-to-participate-in-social_roles-and-activities-4a` |
  | `promis-ability-to-participate-in-social-roles-and-activities-8a` | `promis-ability-to-participate-in-social_roles-and-activities-8a` |
  | `promis-sleep-disturbance-6a`                                     | `sleep-disturbance-6a`                                            |

  The `social_roles` / `social-roles` pair is particularly notable: 2026jan20
  used an underscore in what should be a dash-delimited slug. The 2026apr20
  pipeline sanitizes slugs more consistently.

- **PROMIS pediatric forms are differently organized.** 2026apr20 has 12
  explicit per-population files (`promis-ped-anxiety-8a-ec`, `-proxy`,
  `-self`, `promis-ped-depressive-4a-ec`, `-6a-proxy`, `-8a-self`,
  `promis-ped-global-health-8a-ec`, four `promis-ped-sleep-*` files, and
  `promis-ped-stress-4a`). 2026jan20 has a single consolidated
  `promis-pediatric-anxiety-sf` and no matching files for the others.
  This isn't a rename: 2026apr20 has expanded the PROMIS pediatric
  coverage, while 2026jan20 had different content under a single umbrella.

- **Genuinely new/retired CRFs.** After accounting for renames, some CRFs
  appear truly new in one direction or the other. Examples of CRFs only in
  2026jan20 (dropped from 2026apr20): `addiction-behaviors-checklist`,
  `amps`, `audit-interview`, `audit-self-report`, `cgic`, `cirs`, `csq-24`,
  `fmi`, `gcs`, `gpaq`, `keele-start-back`,
  `microbiome-bristol-stool-chart`, `mypas`,
  `nida-assist-ada-modified-levels-1-2`, `pcs-child`, `pcs-parent`,
  `promis-alcohol-use-7a`, `promis-appeal-of-substance-use-7a`,
  `promis-gastrointestinal-constipation-9a`,
  `promis-neuropathic-pain-quality-5a`,
  `promis-pain-behavior-item-bank-v2.0`,
  `promis-pain-behavior-parent-proxy-item-bank-v1.0`,
  `promis-prescription-pain-med-misuse-7a`,
  `promis-severity-of-substance-use-7a`, `promis-short-form-4a-fatigue`,
  `rmdq`, `socrates`, `taps2`, `timeline-followback`,
  `timeline-followback-90day`, `whodas-2`. 2026apr20 adds CRFs such as
  `cali-9`, `cali-9-proxy`, `capd`, `cds`, `csdc-sf`, `csq50`, `fdi`,
  `jfls`, `maas-a`, `nrs11`, `nscap-rev`, `ort`, `pedmidas`, `pgis-acute`,
  `promis-pf-8c-mod`, `rass`, `s2bi`, `scd-hrss-sf`, `sickle-cell-stress`,
  `stoplight-pain`, `tapes-r`, `umss`.

  These lists are the unmatched remainder after renames — treat them as
  leads to verify, not hard truths.

### 2. Shared files are mostly content-equivalent

139 of 195 shared files (71%) differ only in volatile fields and are
functionally equivalent. The noise comes from:

- `drupal_id` on URL entries — `null` in 2026apr20, numeric strings in
  2026jan20. The 2026apr20 pipeline isn't populating Drupal IDs.
- Timestamped paths in `_sources` fields:
  `Heal_CDE_2026-04-06T130104` (2026apr20) vs
  `Heal_CDE_2026-01-05T130102` (2026jan20) — just reflects the export
  date used.
- NIH CDE document URLs under `/sites/default/files/CDEs/<YYYY-MM>/` —
  the month folder updates on the NIH side.

These are normalized away by the comparison tool before classification.

### 3. Real content changes are concentrated in demographic and mapping-heavy files

56 shared files have record-level changes after normalization. Churn is
heavily skewed toward two files — `pediatric-demographic.dug.json` (5/8/38
records only-A/only-B/changed) and `adult-demographics.dug.json` (3/5/25)
— and a long tail of mapping refreshes. See [`detail.md`](detail.md) for
the full list sorted by churn.

Dominant change categories:

- **Demographic variable renames** (in both adult + pediatric demographics):
  `EDULVL` → `EDULEVEL`, `ETHNIC` → added/reshaped, `GENIDENT`/`GENIDENTOTH`
  added, `HI_LA`/`MENA` added. Many `metadata.question_text`,
  `metadata.short_description`, `metadata.instructions`, and
  `metadata.permissible_values` values are also rewritten. This is a real
  CDE-content change, not a pipeline artifact.
- **Study mapping churn**: the most common changed field across record_changes
  files is `metadata.study_mappings.HDP<id>._sources` and
  `metadata.study_variable_mappings.HDP<id>`. Files like
  `nida-modified-assist` (16 records changed), `patient-health-questionnaire-9`
  (9), `opioid-mme` (5), `taps`, `gad2`, `phq2` are all dominated by
  study-mapping edits — new HDP IDs were added or existing ones gained/lost
  variable mappings.
- **`metadata.categories` changes** appear in ~35 files. 2026apr20 tends
  to collapse categories — e.g. `phq2`: 2026apr20 has
  `Core - Adult & Pediatric`, 2026jan20 has four separate values
  `Adult Acute`, `Adult Chronic`, `All Core`, `Pediatrics`.
- **`parents` shifts** — adding/removing the CRF from a category parent —
  appear in ~25 files, usually alongside a `metadata.study_mappings.HDP*`
  change for the same new study.
- Scattered cosmetic edits to `description`, `metadata.urls`, and
  `metadata.instructions` on a handful of CRFs.

## What this tells us about the two pipelines

- The 2026apr20 pipeline (this repo) produces a **cleaner filename slug**
  (no stray underscores, consistent version suffix placement) and **drops
  `drupal_id` population**. If Drupal IDs are needed downstream, that's a
  regression to investigate.
- The 2026apr20 pipeline **covers more PROMIS pediatric variants**
  explicitly but is **missing a number of CRFs** that 2026jan20 has
  (AUDIT, timeline-followback, several PROMIS substance-use / pain-behavior
  forms, etc.). Worth checking whether these were dropped on purpose or
  fell off the mapping CSVs.
- The **demographic CDEs have been updated** (new gender-identity fields,
  renamed education variable) — that's a real data change the 2026apr20
  run picks up, not a format difference.
- The **study-mapping set has grown** — many CRFs reference new HDP IDs,
  suggesting the HEAL CDE team's REDCap export used for 2026apr20 has more
  study/CRF associations than the 2026jan20 export did.

## Reproducing this report

```bash
uv run python comparators/compare_dug_dirs.py \
    dug-data-model-2026apr20 \
    dug-data-model-2026jan20 \
    reports/2026apr20-from-2026jan20 \
    --label-a 2026apr20 \
    --label-b 2026jan20
```

The tool is directory-generic and works on any two Dug JSON output dirs.
Labels default to directory names; use `--label-a` / `--label-b` for
shorter labels in the generated markdown.
